點餐遊戲 v8_MbyF
================

這是一個為兒童設計的模擬點餐遊戲，支援電腦與手機裝置，並具備廚房同步顯示功能。系統使用 Firebase 作為資料儲存後端，並支援 PWA 離線使用與安裝。

📁 檔案說明
----------

1. index.html
   - 桌機版主畫面，預設進入模式為「點餐模式」
   - 可切換至「編輯模式」進行分類與餐點設定

2. mobile.html
   - 手機版專用畫面，自動由 index.html 導向
   - 精簡版點餐體驗，包含分類、選餐、已點清單與送出訂單

3. kitchen.html
   - 廚房模式，用於顯示即時訂單
   - 可選擇不同「分店」（即菜單名稱），並查看、完成訂單

4. script.js
   - 所有頁面共用的邏輯核心
   - 包含：分類與餐點管理、點餐流程、Firebase 同步邏輯、訂單送出

5. style.css
   - 桌機與手機通用樣式
   - 包含：分類按鈕、餐點卡片、編輯介面、送出按鈕等樣式定義

6. manifest.json
   - Progressive Web App 設定檔
   - 定義名稱、圖示、起始頁、主題顏色等資訊

7. service-worker.js
   - PWA 快取設定，支援離線使用
   - 預先快取 index.html、CSS、JS、icon 圖示

🔧 功能特色
----------

✅ 點餐功能
- 餐點可自訂分類（顏色 + 文字顏色）
- 支援「一般價格」與「大份價格」
- 顧客可自由點選，即時計算總金額
- 支援移除已點項目

✅ 編輯模式
- 可新增、刪除分類
- 可新增、刪除餐點
- 可命名並儲存多份菜單
- 菜單儲存於 Firebase 雲端資料庫

✅ 廚房即時接單
- 可選擇要查看的「分店」（對應菜單名稱）
- 即時接收新訂單（自 Firebase）
- 可標記訂單為已完成並刪除

✅ 手機支援
- 自動導向 mobile.html，介面簡潔
- 僅顯示顧客點餐所需項目
- 手機點餐與桌機共用資料邏輯

✅ Firebase 整合
- 實作菜單儲存 (menus/{menuName})
- 訂單儲存至對應分店 (orders/{menuName})

✅ PWA 支援
- 支援離線使用與主畫面安裝
- 使用 manifest + service-worker 設定

📦 Firebase 設定
-------------

Firebase 專案 ID：order-game-3a2c3

Realtime Database 路徑說明：
- /menus/{menuName}：儲存分類與餐點
- /orders/{menuName}：儲存該菜單對應的訂單清單

📲 使用方式
----------

1. 開啟 index.html（電腦版）或 mobile.html（手機）
2. 在編輯模式中新增分類與餐點，並儲存為某個菜單名稱
3. 回到點餐畫面，選擇該菜單後進行點餐
4. 廚房可透過 kitchen.html 選擇相同菜單名稱，即時查看訂單顯示情況

📌 注意事項
----------

- 若要刪除分類，請先刪除該分類下所有餐點
- 菜單名稱會同步用作「分店名稱」用於訂單分流
- 每次送出訂單時，會將內容與時間寫入 Firebase 對應路徑
- 支援跨裝置、跨畫面同步資料，適合雙 iPad 模式使用
